Setup

  1. Run Simulation: Start ../UnitySimulation/Environment Samsung.exe (see UnitySimulation README). The simulation must be running for the website to connect via WebSocket.
  2. Serve Website: Navigate to the Website directory in your terminal and start a simple web server (requires Python 3): 
    cd Website
python -m http.server 8000
    (If you don't have Python, use another local server like Node.js http-server)
  3. Open Browser: Visit http://localhost:8000 in a compatible browser (Chrome/Edge recommended for voice features).

Usage

  1. Start: Go to this landing page (/index.html).
  2. Navigate: Click the "Control the Simulation" link above, which takes you to /controller/index.html.
  3. Launch Controller: Click the link on the chooser page to open the main /controller/v1.
  4. Control:
    • Use buttons, sliders, and inputs for direct manual control of devices.
    • Type commands into the text input (e.g., "Turn on the living room light") and press Enter or click Send.
    • Click the microphone icon (ðŸŽ¤) to issue voice commands (requires microphone permission).
  5. Check Status: The connection status ("Connected" / "Disconnected") is shown at the top right of the controller page.

Core Features

  • Real-time Control: Connects to the simulation via WebSocket (ws://localhost:8080/iot).
  • Voice Commands: Utilizes browser's Speech Recognition API (best support in Chrome/Edge). Ensure microphone access is granted.
  • Text Commands: Send natural language commands for AI processing.
  • Manual Overrides: Directly manipulate device states via UI controls.
  • Keyboard Shortcuts:
    • Ctrl + /: Focus the text command input field.
    • Alt + V: Start/stop voice command recognition.
    • Ctrl + R: Attempt to reconnect WebSocket (prevents full page reload).

Troubleshooting

Issue Fix
Controls unresponsive Ensure the Unity simulation is running and WebSocket server is active.
"Disconnected" status Check simulation status. Verify WebSocket URL (ws://localhost:8080/iot) and browser console for errors.
Voice not working Use Chrome/Edge, allow microphone permission, and check the console for errors.
WebSocket errors Examine browser console (F12) for error messages and verify the server URL.
Layout issues Hard refresh (Ctrl+F5 or Cmd+Shift+R) and clear browser cache.
Command errors Review the browser console for errors from main.js or WebSocket messages.

Development

  • Styles: Modify assets/css/main.css (for the controller page).
  • Logic: Update assets/js/main.js (includes WebSocket, UI interactions, and device controls).
  • HTML Structure: Edit controller in the controller/ directory or this main index.html.
  • API: Refer to the UnitySimulation API for details on WebSocket command structure.
  • Contribute: Follow guidelines in CONTRIBUTING.md.
